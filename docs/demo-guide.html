1  — Quick Start Get everything running in 4 steps
Phase 0 — UI Only The current phase is a fully interactive React UI simulation. No real backend is required. Steps 1 and 2 below (Docker + Spring Boot) apply to full-stack testing. For UI-only demos, skip to Step 3.
Step 1 — Start Infrastructure (Docker)
# From the project root — starts PostgreSQL, Keycloak, Mailhog, Redis
$ docker compose up -d

# Verify all containers are healthy
$ docker compose ps
Step 2 — Start Backend (Spring Boot)
$ cd backend
$ mvn spring-boot:run -Dspring-boot.run.profiles=dev

# Backend ready when you see:
# Started PrimusApplication in X.XXX seconds (JVM running for X.XXX)
# API available at http://localhost:8080
Step 3 — Start Provider Portal
$ cd apps/provider-portal
$ npm install       # first time only
$ npm run dev

# Running at http://localhost:5173
Step 4 — Start Patient Portal
$ cd apps/patient-portal
$ npm install       # first time only
$ npm run dev

# Running at http://localhost:5174
Ready to demo! Provider Portal: http://localhost:5173  |  Patient Portal: http://localhost:5174  |  Backend API: http://localhost:8080

  9. 9In the Assessment & Plan column, type a diagnosis name or ICD-10 code — show the autocomplete dropdown with descriptions. Add a diagnosis

5  — Troubleshooting Common issues and fixes
Docker containers fail to start or crash immediately
Check Docker Desktop is running — it must be open before docker compose up.
Free up port conflicts: PostgreSQL uses :5432, Keycloak uses :8180, Redis uses :6379.
Check for conflicts: lsof -i :5432 and kill any blocking processes.
Low memory: Allocate at least 6 GB RAM to Docker Desktop in Settings → Resources. 
# Reset all containers and volumes (last resort)
$ docker compose down -v
$ docker compose up -d
Port 5173 or 5174 already in use — Vite fails to start
Another Vite dev server is likely running. Kill it: 
$ lsof -ti:5173 | xargs kill -9
$ lsof -ti:5174 | xargs kill -9
Or start on a different port: npm run dev -- --port 5175
Spring Boot fails: "Unable to acquire JDBC Connection" or "Connection refused"
PostgreSQL container is not ready yet. Wait 15–20 seconds after docker compose up -d before starting the backend. 
# Verify PostgreSQL is accepting connections
$ docker compose exec postgres pg_isready -U primus
# Should output: /var/run/postgresql:5432 - accepting connections
npm install fails with ERESOLVE or peer dependency errors
Node.js version mismatch. Primus requires Node 20+. 
$ node -v   # must be v20.x.x or higher

# If using nvm:
$ nvm install 20 && nvm use 20

# Then retry:
$ rm -rf node_modules package-lock.json
$ npm install
Login fails or redirects to Keycloak with an error
In Phase 0 (UI-only), login uses mock authentication — no real Keycloak call is made. If you see a Keycloak redirect, the portal is mistakenly pointing to a real auth endpoint.

Check that VITE_USE_MOCK_AUTH=true is set in apps/provider-portal/.env.development and apps/patient-portal/.env.development. 
# .env.development
VITE_USE_MOCK_AUTH=true
VITE_API_BASE_URL=http://localhost:8080
Maven build fails: "Java version 17 required but 21 found" or similar
Primus backend requires Java 21. Check your active JDK: 
$ java -version
# Should show: openjdk 21.x.x

# Switch JDK with SDKMAN:
$ sdk use java 21.0.3-tem
Page shows blank screen or React error boundary in browser
Open the browser DevTools console (F12) and look for the specific error. Common causes:
  • Missing env variable: Copy .env.example to .env.development in the portal directory.
  • Stale cache: Hard reload with Ctrl+Shift+R (or Cmd+Shift+R on Mac).
  • Module not found: Run npm install again — a dependency may be missing.
  • TypeScript error: Run npm run type-check to see all type errors before starting dev server.
Swagger UI shows no endpoints or "Failed to fetch" errors
The Spring Boot backend must be running. Check: 
# Verify backend is up
$ curl http://localhost:8080/actuator/health
# Expected: {"status":"UP",...}
If the health check returns a connection error, the backend process is not running — restart it with mvn spring-boot:run -Dspring-boot.run.profiles=dev.
Still stuck? Run docker compose logs --tail=50 to see container logs. For backend issues, check backend/logs/application.log. For frontend issues, check the browser DevTools console. Contact the Thinkitive team at dev@thinkitive.com.
